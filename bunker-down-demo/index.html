// key_features[]
◈// feature_01.png
Card & Deck System

Scriptable object-based card architecture with runtime deck management. Cards define their own data, allowing decks to be authored and modified without code changes - designed for extensibility across card types and game modes.

◈// feature_02.png
Game Phase Manager

Centralised game manager driving turn structure and phase sequencing. Coordinates state transitions across draw, play, and resolution phases, with hooks for NPC turns and win condition evaluation.

◈// feature_03.png
Animated UI Game Board

All player interaction delivered through a single physical-style button board, emulating the tactile feel of a tabletop game. UI state and transitions driven entirely by tweens, with board layout reflecting current game phase.

◈// feature_04.png
NPC Opponents

Four opponents with distinct behavioural profiles - each making independent card and resource decisions based on game state. Shared underlying systems with per-NPC logic layered on top to produce meaningfully different playstyles.
